Pairing Your Nimbus Server and Nimbus Client

Nimbus Server is configured to work out of the box, although if you are using Nimbus Client or want to access Nimbus Server from your laptop (host) PC, you need to change the default Nimbus Server IP address in the Nimbus Client or the host machine hosts file to the actual IP address of your Nimbus Server VM.  To determine your Nimbus Server IP address, run the ifconfig command from a terminal window and use the IP address assigned to the ens33 (NAT) virtual network adapter if running on VMWare, or eth0 if running on AWS. Otherwise, no changes are necessary.  You should not change the IP address of the Nimbus Server host name.

The username is demo and the password is Password1.  If you need elevated root privileges, the root password is also Password1.

Performance Tweaking

Nimbus Server was configured with default VMware Workstation Virtual Machine Settings that work for the majority of employees, but you should adjust the Memory and Processors settings to reflect your actual hardware.  Failure to do so will almost certainly affect virtual machine performance.

Downloading Docker Images

Nimbus Server contains no Docker images upon initial download.  You need to pull (download) demo images that meet your from Docker Hub. For using docker-app or nimbusapp, please select an image ending in .dockerapp.

Repository Information

The image repositories on Docker Hub have tabbed sections, two of which are described below:

  • Overview: Contains a description of the application, shows the correct nimbusapp command to pull the images and create the running containers, and provides container access credentials and URLs.
  • Tags: Lists the available image versions, which are equal to the Micro Focus product version numbers where applicable.

nimbusapp

This version of Nimbus Server comes with a script called nimbusapp designed to help leverage the power of Docker Compose and Docker App to help get your Nimbus Docker containers started easier. In order to learn more about it, please type “nimbusapp --help” in a terminal session.

Docker Login

In order to use private repositories, and remove certain rate limits on Docker downloads, you may wish to log in to Docker Hub on the command line. To do so, use the docker login command and supply your Docker Hub credentials.

Docker Behind A Proxy

This version of Nimbus Server comes with a script called nimbus-docker-proxy designed to help add proxy settings to your Docker daemon if you are behind a proxy. In order to learn more about it, please type "nimbus-docker-proxy --help" in a terminal session.

Docker Images from FTP

Employees may need to download some images from a private FTP site, which can be accomplished with the nimbus-ftp-load command.
