Pairing Your Nimbus Server and Nimbus Client

Nimbus Server is configured to work out of the box, although if you are using Nimbus Client or want to access Nimbus Server from your laptop (host) PC, you need to change the default Nimbus Server IP address in the Nimbus Client or the host machine hosts file to the actual IP address of your Nimbus Server VM.  To determine your Nimbus Server IP address, run the ifconfig command from a terminal window and use the IP address assigned to the ens33 (host-only) virtual network adapter if running on VMWare, or eth0 if running on AWS. Otherwise, no changes are necessary.  You should not change the IP address of the Nimbus Server host name.

The username is demo and the password is Password1.  If you need elevated root privileges, the root password is also Password1.

Performance Tweaking

Nimbus Server was configured with default VMware Workstation Virtual Machine Settings that work for the majority of employees, but you should adjust the Memory and Processors settings to reflect your actual hardware.  Failure to do so will almost certainly affect virtual machine performance.

Downloading Docker Images

Nimbus Server contains no Docker images upon initial download.  You need to pull (download) demo images that meet your needs after first logging in to Docker Hub.  From a Linux terminal window, use the docker login command and provide your Docker ID and password.  You can also log in to Docker Hub from a browser to view available images and their details.  Please contact Keith Cutler if you need access to the private admpresales organization on Docker Hub.

Repository Information

Available Docker Images are visible by going to https://hub.docker.com/u/admpresales/dashboard/

The image repositories on Docker Hub have tabbed sections, two of which are described below:

  • Repo Info: Contains a description of the image, shows the correct docker run command to pull the image and create the running container, and provides container access credentials.
  • Tags: Lists the available image versions, which are equal to the Micro Focus product version numbers where applicable.  Tags may also indicate whether demo data is included, whether integrations are included and so forth.

Docker Run Command

Docker run commands are provided in the Repo Info tab and you can copy and paste them into a terminal window for ease of use.  Remember to supply the appropriate tag for the image you wish to run.  Docker run commands pull images if they don’t exist locally and create running containers.

nimbusapp

This version of Nimbus Server comes with a script called nimbusapp designed to help leverage the power of Docker Compose and Docker App to help get your Nimbus Docker containers started easier. In order to learn more about it, please type “nimbusapp --help” in a terminal session.

Docker Behind Proxy

This version of Nimbus Server comes with a script called docker-proxy designed to help add proxy setting to your Docker daemon if you are behind a proxy. In order to learn more about it, please type "docker-proxy --help" in a terminal session.
